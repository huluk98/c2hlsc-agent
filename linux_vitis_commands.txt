Linux Vitis run commands
========================

1. Pull the latest fixes

cd c2hlsc-agent
git pull


2. Activate your existing Conda environment

conda activate hlsc
python -m pip install -r requirements.txt
python -m pip install -e .


3. Put your vitis_hls binary path in the local path file

nano vitis_hls_bin_path.txt

Put exactly one line in the file, for example:

/path/to/Vitis_HLS/2024.2/bin/vitis_hls

Save and exit nano:

Ctrl+O
Enter
Ctrl+X


4. Clean the old failed output

rm -rf build/vector_add


5. Run the Vitis flow

python scripts/run_vitis_with_bin.py \
  --config examples/vector_add/config.yaml \
  --out build/vector_add \
  --print-env

This command works because scripts/run_vitis_with_bin.py reads the path from
vitis_hls_bin_path.txt. The file is intentionally ignored by git so your local
machine path does not get committed.


6. Check the conversion report

cat build/vector_add/conversion_report.md

You want to see:

Software equivalence: pass
C simulation: pass
C synthesis: pass
C/RTL co-simulation: pass


7. Find generated RTL files

find build/vector_add -type f \( -name "*.v" -o -name "*.sv" -o -name "*.vhd" \)

Likely RTL folder:

ls build/vector_add/c2hlsc_project/solution1/syn/verilog


8. If CSim or Vitis still fails, collect debug info

cat build/vector_add/run_hls.tcl
cat build/vector_add/run_csim.tcl
cat build/vector_add/run_csynth.tcl
cat build/vector_add/run_cosim.tcl
tail -120 build/vector_add/csim.log
tail -120 build/vector_add/csynth.log
tail -120 build/vector_add/cosim.log
find build/vector_add/c2hlsc_project -type f -name "*.log" -print


Alternative: pass the binary path on the command line without editing the Python file
================================================================================

rm -rf build/vector_add

python scripts/run_vitis_with_bin.py \
  --vitis-hls-bin "/path/to/Vitis_HLS/2024.2/bin/vitis_hls" \
  --config examples/vector_add/config.yaml \
  --out build/vector_add \
  --print-env


Run the single embedded JSON smoke bundle
=========================================

Use this when you only want to check that your local Vitis can run CSim,
CSynth, and C/RTL CoSim on a small known-good HLS-C/testbench bundle:

python scripts/run_vitis_bundle.py \
  --vitis "/path/to/Vitis_HLS/2024.2/bin/vitis_hls"

The runner unpacks configs/simple_calculator_vitis_cosim_bundle.json into
build/vitis_bundle_run and writes build/vitis_bundle_run/vitis_hls.log.


Run the accepted JSONL dataset and verify Verilog generation
============================================================

Use this for dataset rows such as hls_nl_repaired.accepted.jsonl. The runner
generates one Vitis project folder per JSONL row, runs CSim + CSynth, and marks
a row pass only when Vitis exits successfully and emits .v/.sv files under:

hls_nl_project/solution1/syn/verilog

Small smoke check:

VITIS_HLS_BIN="/path/to/Vitis_HLS/2024.2/bin/vitis_hls" \
python scripts/run_hls_nl_vitis_batch.py \
  --input /path/to/hls_nl_repaired.accepted.jsonl \
  --out-dir build/hls_nl_accepted_vitis_smoke \
  --limit 10 \
  --part xczu7ev-ffvc1156-2-e \
  --clock 10

Full accepted JSONL run:

VITIS_HLS_BIN="/path/to/Vitis_HLS/2024.2/bin/vitis_hls" \
python scripts/run_hls_nl_vitis_batch.py \
  --input /path/to/hls_nl_repaired.accepted.jsonl \
  --out-dir build/hls_nl_accepted_vitis_full \
  --part xczu7ev-ffvc1156-2-e \
  --clock 10

Check summary:

python - <<'PY'
import json
from pathlib import Path
r = json.loads(Path("build/hls_nl_accepted_vitis_smoke/vitis_verilog_report.json").read_text())
print(json.dumps(r["summary"], indent=2))
PY

For C/RTL CoSim too, add:

--run-full-cosim


Export only full-CoSim passes for upload
========================================

Run the full batch in CoSim mode:

VITIS_HLS_BIN="/path/to/Vitis_HLS/2024.2/bin/vitis_hls" \
python scripts/run_hls_nl_vitis_batch.py \
  --input /path/to/hls_nl_repaired.accepted.jsonl \
  --out-dir build/hls_nl_accepted_full_cosim \
  --part xczu7ev-ffvc1156-2-e \
  --clock 10 \
  --run-full-cosim

Then export only passing cases into a clean uploadable corpus:

python scripts/export_cosim_successes.py \
  --report build/hls_nl_accepted_full_cosim/vitis_batch_report.json \
  --out-dir hls_nl_full_cosim_passes

The export keeps minimal per-case files plus cosim_pass_evidence.json. Failed
and skipped rows stay in failed.jsonl and skipped.jsonl for inspection.


Small JSONL CoSim smoke before scaling up
========================================

This uses configs/hls_nl_cosim_smoke.json. By default it runs one accepted JSONL
row through CSim, CSynth, and C/RTL CoSim, then prints the Vitis log tail,
generated Verilog paths, and CoSim artifact paths.

cd c2hlsc_agent

VITIS_HLS_BIN="/path/to/Vitis_HLS/2024.2/bin/vitis_hls" \
HLS_NL_JSONL="/path/to/hls_nl_repaired.accepted.jsonl" \
  bash scripts/run_hls_nl_cosim_smoke.sh

Useful overrides:

HLS_NL_LIMIT=3
HLS_NL_OFFSET=100
HLS_NL_OUT_DIR=build/hls_nl_cosim_offset100
VITIS_PART=xczu7ev-ffvc1156-2-e
VITIS_CLOCK=10

Generate projects without running Vitis:

HLS_NL_GENERATE_ONLY=1 \
HLS_NL_JSONL="/path/to/hls_nl_repaired.accepted.jsonl" \
  bash scripts/run_hls_nl_cosim_smoke.sh
